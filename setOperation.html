Set Operations

In this lesson, we will learn and visualize some operations in set, unordered_set, and multiset.

  • insert(value): Inserts a value into the set or multiset. In set and unordered_set, duplicates are ignored. In multiset, duplicates are allowed. Time complexities are O(log n) for set and multiset, O(1) for unordered_set.
  • emplace(value): Works similarly to insert(), but can be more efficient by constructing in-place. Time complexity is the same as insert(): O(log n) for set and multiset, O(1) for unordered_set.
  • count(value): Returns how many times a value exists in the set. For set and unordered_set, this will be either 0 or 1. For multiset, it returns the number of occurrences. Time complexities are O(log n) for set/multiset, O(1) for unordered_set.
  • find(value): Checks if a value exists in the set. Returns an iterator to the value if found, otherwise returns end(). Time complexity is O(log n) for set and multiset, O(1) for unordered_set.
  • erase(value): Removes all instances of the value in set and unordered_set. In multiset, only one occurrence is removed at a time. Time complexities are O(log n) for set/multiset, O(1) for unordered_set.
  • lower_bound(value): Returns an iterator to the first element that is not less than the given value. Only works in set and multiset since they are ordered. Not available in unordered_set. Time complexity: O(log n).
  • upper_bound(value): Returns an iterator to the first element that is greater than the given value. Like lower_bound, only available for set and multiset. Not supported in unordered_set. Time complexity: O(log n).
