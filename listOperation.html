List Operations

Lists provide a set of operations that make it easy to manage collections of data. Here, we'll explore six key operations: `push_back`, `push_front`, `pop_back`, `pop_front`, `insert`, and `access`.

ðŸ”‘ List functions:

  • push_back(): Adds an element to the end of the list.
    Time Complexity: O(1)
  • push_front(): Adds an element to the beginning of the list.
    Time Complexity: O(1)
  • pop_back(): Removes the last element from the list.
    Time Complexity: O(1)
  • pop_front(): Removes the first element from the list.
    Time Complexity: O(1)
  • insert(position,val): Inserts element at a specified position in the list.
    Time Complexity: O(n)
  • remove(val):Removes all element if matches with value.
    Time Complexity: O(n)
  • merge(l1,l2):Merge two sorted list.
    Time Complexity: O(n+m)

ðŸ“Œ Key Takeaways:

  • Lists provide frequent insertion and removal of elements at any position which makes it different from other sequential containers like vector and deque.
  • Insertion at arbitrary positions and accessing elements have a higher time complexity due to the need to traverse the list.
  • Lists are a good choice when we need to frequently add or remove elements at the beginning or end, but may not be the best choice when you need to frequently access, inswert, or remove elements.
