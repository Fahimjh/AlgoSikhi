Queue Operations

In this section, we'll visualize key operations in queue and priority_queue.

üîÅ Queue Operations:

  • enqueue(value): Adds an element to the rear of the queue.
    ‚è± Time Complexity: O(1)
  • dequeue(): Removes an element from the front of the queue.
    ‚è± Time Complexity: O(1)
  • front(): Returns the front element without removing it.
    ‚è± Time Complexity: O(1)
  • rear(): Returns the last (rear) element.
    ‚è± Time Complexity: O(1)
  • empty(): Checks if the queue is empty.
    ‚è± Time Complexity: O(1)
  • size(): Returns the current number of elements in the queue.
    ‚è± Time Complexity: O(1)

üìå Differences from Stack:

  • Queue follows FIFO, Stack follows LIFO
  • In Queue, you insert at rear and remove from front
  • In Stack, both insertion and removal happen at the top

Priority Queue Operations

A priority queue always removes the element with the highest priority first. Internally, it's usually implemented with a heap.

üîÅ Priority Queue Operations:

  • push(value): Adds an element to the priority queue.
    ‚è± Time Complexity: O(log n)
  • pop(): Removes the element with the highest priority.
    ‚è± Time Complexity: O(log n)
  • top(): Returns the element with the highest priority.
    ‚è± Time Complexity: O(1)
  • empty(): Checks if the priority queue is empty.
    ‚è± Time Complexity: O(1)
  • size(): Returns the number of elements in the priority queue.
    ‚è± Time Complexity: O(1)

üìå Differences from Queue:

  • Priority Queue serves elements based on priority, not arrival order
  • Queue always serves in order of arrival (FIFO)
  • In Priority Queue, elements may not maintain insertion order
