Stack Operations

In this lesson, we will learn and visualize some common operations on a stack. A stack follows the LIFO (Last In, First Out) principle — the last element pushed is the first to be popped.

  • push(value): Adds an element to the top of the stack.
    Time Complexity: O(1)
  • pop(): Removes the top element from the stack.
    Time Complexity: O(1)
    ⚠️ Be careful to check if the stack is empty before popping.
  • top(): Returns the top element of the stack without removing it.
    Time Complexity: O(1)
  • empty(): Returns true if the stack is empty, otherwise false.
    Time Complexity: O(1)
  • size(): Returns the total number of elements currently in the stack.
    Time Complexity: O(1)
  • swap(): Swaps the contents of two stacks.
    Time Complexity: O(1)

✅ Stack Use Cases:

  • Undo/Redo functionality in editors
  • Backtracking (e.g., maze solvers, recursive calls)
  • Expression evaluation and syntax parsing
  • Function call management via call stack

⚠️ Important Note:

  • Stack does not allow random access — you can only access the top.
  • Trying to pop or view the top of an empty stack may cause errors.
