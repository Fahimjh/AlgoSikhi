Introduction to Deque

Deques are double-ended queues that allow efficient insertion and removal of elements from both ends. They works like dynamic array, thats why their elements can be accessed directly using an index.

ðŸ”‘ Key Characteristics:

  • Double-Ended: Elements can be added or removed from both the front and back of the deque.
  • Non-Contiguous Memory: Elements are not stored in consecutive memory locations, unlike vectors or arrays.
  • Efficient Insertion/Removal: Deques allow efficient insertion and removal of elements from both ends, making them suitable for applications with frequent additions or removals.
  • Random Access: Deques provide fast access to elements using their index, similar to vectors.
  • assign(count, value): We can assigns new values to the deque also like vector, replacing old ones.
    Time Complexity: O(n)

ðŸ“Œ Common Use Cases:

  • Implementing queues or stacks with efficient insertion and removal from both ends.
  • Storing collections of data that need frequent additions or removals from both ends.
  • Useful in applications with limited memory, as deques do not require contiguous memory allocation.
