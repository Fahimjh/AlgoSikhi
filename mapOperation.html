map Operations

In this lesson, we will learn and visualize some operations in map, unordered_map and multimap.

  • insert({key,value}): Inserts a new sorted key-value pair into the map and multimap. In unordered_map, the key-value pair is inserted at the end of the map as they are not sorted. Time complexities for this operation are O(logn) for map and multimap. O(1) for unordered_map.
  • emplace(key,value): Inserts a new sorted key-value pair into the map and multimap. In unordered_map, the key-value pair is inserted at the end of the map as they are not sorted. Time complexities for this operation are O(logn) for map and multimap. O(1) for unordered_map.
  • count(key): Counts the number of occurrences of a specific key in the unordered_map and multimap. As map desn't allow duplicate keys, count(k) always returns 1 in map. Time complexities for this operation are O(logn) for map and multimap. O(1) for unordered_map.
  • find(key): Finds the if a key is present in all types of map. It returns iterator to the element if present else returns end iterator if finds the key. Time complexities for this operation are O(logn) for map and multimap. O(1) for unordered_map.
  • erase(key): Removes the key with value if its/they present in all types of map. Time complexities for this operation are O(logn) for map and multimap. O(1) for unordered_map.
